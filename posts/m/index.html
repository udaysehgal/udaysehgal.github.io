Installing Docker

Installing the repository

Before we install Docker, we need to set up the Docker repository. Later, we will update Docker from this repository.

First, we update the apt package index through the first command & then we install packages to allow apt to use a repository

sudo apt-get update

sudo apt-get install \
apt-transport-https \
ca-certificates \
curl \
gnupg \
lsb-release

Then we add Dockerâ€™s official GPG key:

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

GPG stands for Gnu Privacy Guard. It is useful for creating public and private keys. These can be used for digital signatures or encryption/decryption.

The next command is used to set up the stable repository.

echo \
  "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
Installing Docker Engine

First, we update the apt package index & then install docker & containerd.

sudo apt-get update

sudo apt-get install docker-ce docker-ce-cli containerd.io

We can also install a specific version of Docker Engine by using the following command:

apt-cache madison docker-ce

After typing this command, we got the following output:

docker-ce | 5:20.10.7~3-0~ubuntu-hirsute | https://download.docker.com/linux/ubuntu hirsute/stable amd64 Packages
docker-ce | 5:20.10.6~3-0~ubuntu-hirsute | https://download.docker.com/linux/ubuntu hirsute/stable amd64 Packages

We can install a specific version using the version string (the text after the first |). The code for that installation is:

sudo apt-get install docker-ce=<ver_str> docker-ce-cli=<ver_str> containerd.io

where ver_str is the version string

When we tried to do that, it did not successfully run as we had already installed Docker through the first step.

Now, to check that Docker has been installed, run the following:

sudo docker run hello-world

The output should be a message after it downloads a test image from docker & runs it in the container.
