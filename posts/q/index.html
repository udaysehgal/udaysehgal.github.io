Creating a VM in AWS using Terraform

We will now create a file using Terraform that defines a single AWS EC2 instance.

First, we make a new directory and change into it to make it our working directory

mkdir terraform-aws
cd terraform-aws

We now create a new .tf file inside this

touch main.tf

We open the file & paste the following code inside it:

terraform {  
  required_providers {   
   aws = {      
    source  = "hashicorp/aws"    
    version = "~> 3.27"    
    }
  }
  required_version = ">= 0.14.9"}

provider "aws" { 
  profile = "default"  
  region  = "us-west-2"
 }

resource "aws_instance" "app_server" {  
  ami           = "ami-830c94e3"  
  instance_type = "t2.micro"
   
  tags = {    
  Name = "ExampleAppServerInstance"  
  }
 }

Next, we initialise the directory with the command

terraform init

Terraform then downloads the aws provider and installs it in a subdirectory of the current working directory.

Next, we need to format and validate the configuration,

terraform fmt

The command updates configurations in the current directory. If the file was already formatted correctly, Terraform won’t return any output. Otherwise, it would print out the names of the files it modified after the command was used.

In this case, no output is returned as the file was already formatted correctly.

Next, we validate the configuration by using

terraform validate

If the configuration is valid, Terraform will return a message confirming it.

Now, we just have to apply the following command,

terraform apply

Terraform will print out an execution plan. Type yes to proceed.

The EC2 instance has been created.
