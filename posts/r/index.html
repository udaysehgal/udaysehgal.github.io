Tasks for today: Using AWS Cli for creating a VPC

We have already worked & documented creating an EC2 instance on the Amazon cloud server earlier. We now move on to using the client for creating a VPC.

A VPC (Virtual Private Cloud) is basically a private virtual network which lets us launch our resources inside it. It is useful for providing a sort of isolation from the devices on the original public cloud server (eg.AWS in this case)

First, we will try running a VPC manually & then using Terraform. Later, we will also create an instance inside our VPC.

VPC using the AWS Console

  • The first thing that we do is open the Amazon VPC console (https://console.aws.amazon.com/vpc/). We need to select a VPC Configuration. We will be choosing the VPC with a Single Public Subnet option. ublic subnet instances use Public IPs to access the Internet. It uses a /16 network with a /24 subnet.

/16 refers to a 16 IPv4 CIDR block (a network with 65,536 private IP addresses) while

/24 refers to a size /24 IPv4 subnet (a range of 256 private IP addresses)

  • The next step is to name the VPC & the subnet. We would not be changing any of the autofilled data except for the name of the VPC & the Subnet.

  • We click on create VPC and the VPC should be successfully created!

Creating an instance inside our VPC

Now, we move on to launching an instance inside the VPC we have just created.

  • We open the amazon VPC console and click on Launch Instance.
  • Next, we have to choose an AMI. In this case, we choose Amazon Linux 2 AMI (HVM), SSD Volume Type - ami-0721c9af7b9b75114 (64-bit x86)
  • Now, we choose the Instance type which in this case is t2.micro. Click on Review & Launch
  • On the Configure Instance Details page, we need select the VPC and the subnet that we created. We leave the rest of the settings unchanged & move on.
  • Optionally, we can add a name to our instance through the Add Tags page. This helps us identify our instance later. This does not affect the functioning of the instance.
  • We do not need to change any settings on the Configure Security Group page.
  • Next, on the Review Instance Launch page, we choose Launch.
  • A dialog box appears that says “Select an existing key pair or create a new key pair”. We will create a new key pair and download it to our device. This will be required to connect to our instance after it has been launched.
  • We can click on View Instances on the Instances page to see that our instance has been successfully launched.

We get an error that says the instance cannot be launched as the t2.micro is not available in the region we are working on. We need to change our region and create a new VPC there as this one won’t work for the other region.

After we do that, we get a confirmation message.

We have successfully created an instance inside our own VPC!

Next, we need to stop/terminate the instance as well as delete the VPC to ensure the AWS resources aren’t wasted.

This shows us that the instance we ran was inside the VPC. (The VPC ID & the Subnet ID in the image match the original IDs)

Our next step would be automating this whole process using Terraform.
